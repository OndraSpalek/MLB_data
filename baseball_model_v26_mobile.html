🆕 Novinky modelu v9
Motivace: analýza 69 hist. sázek (duben 2026) ukázala systematické overconfidence na silných favoritech — model v8 produkoval predikce 75–87 % u čtvrtiny sázek, přičemž skutečná win rate v tomto pásmu byla jen 35 %.
VRSTVA 1
Bayesovský shrinkage statistik nadhazovačů

V dubnu má pitcher 10–20 IP. SIERA 2.10 po 3 startech je z 70 % šum, ne talent. Každá statistika se proto regreduje k prior hodnotě (ligový průměr nebo preseason projekce) pomocí vzorce:

regressed_stat = (IP × observed + K × prior) / (IP + K)
SIERA
K = 40
K-BB%
K = 50
xFIP
K = 50
FIP
K = 60
BABIP
K = 100

wRC+ regrese: regressed_wRC = (PA × wRC + 200 × 100) / (PA + 200). Týmová ofenzíva se stabilizuje ~200 PA na battera.

Platoon wRC+ splits (v10): Model vybírá wRC+ soupeřovy pálky podle ruky startujícího nadhazovače – LHP startér → wRC+ soupeře vs LHP, RHP startér → wRC+ soupeře vs RHP. Split se zamrazí na startéra a drží se i po nasazení bullpenu.

VRSTVA 2
Negative Binomial místo Poisson

Poisson předpokládá variance = mean. Ale skutečný MLB run scoring má variance ≈ 1,3× mean. Negative Binomial (Gamma-Poisson mixture) tuto přebytečnou rozptyl modeluje přesněji:

r = λ / (dispersion − 1) · · p = 1 / dispersion
γ ∼ Gamma(r, (1−p)/p) · · · runs ∼ Poisson(γ)

Výsledek: vyšší pravděpodobnost extrémních výsledků (shutout nebo exploze) → menší jistota o favoritu.

VRSTVA 3
Logit shrinkage + kontextuální cap

Výsledná raw pravděpodobnost se kalibruje ve dvou krocích:

logit = log(p / (1−p))
shrunk = 1 / (1 + exp(−logit × k)) · · k = 0,80
Kontextuální cap
Dny 1–20: cap 62 % · Dny 21–44: cap 65 % · Dny 45+: cap 68 % · TBD starter: cap 60 % · Divisional game: −2 % · Playoff: cap 70 %
Empirické zdůvodnění
V MLB regulérní sezoně méně než 1 % zápasů má tržní implied prob nad 70 % (Vegas data). Model v8 produkoval 75–87 % u 25 % sázek.
📋 True Talent Score (v10)

Vážené True Talent skóre (v10): Skóre = SIERA×0,40 + kbbERA×0,30 + xFIP×0,20 + FIP×0,10

K-BB% se transformuje na ERA-ekvivalent: kbbERA = 6,25 − 0,15 × K-BB% (lineární mapování: 15 % → 4,00 · 25 % → 2,50 · 5 % → 5,50).

BABIP korekce ±5 % · Normalizace na λ = skóre × (4,50 / 4,20) · Bullpen 40 % váha · wRC+ platoon úprava · Park Factor · +0,15 HFA · 10 000× NegBin simulace.

📈 Vyhodnocení tipy
—
Analýza historie tipů uložených v záložce Tipy dle modelu. Doplň výsledky zápasů (manuálně nebo automaticky přes MLB API) a uvidíš metriky: Hit Rate, Brier Score, Log Loss, ROI a kalibrační graf.
ℹ️ Co která metrika znamená? (klikni pro rozbalení)
Hit Rate — kolik procent tvých tipů skončilo výhrou. U výběru z dvou stran je 50 % náhoda; ~54 % je domácí baseline; 55 %+ začíná dávat smysl.
Brier Score — kvadratická chyba pravděpodobnosti. Když model řekne 70 % a tým vyhraje, chyba je (1 − 0,70)² = 0,09; když prohraje, chyba je (0 − 0,70)² = 0,49. Sčítá se přes všechny tipy a dělí počtem. Nižší = lepší. Náhoda 50/50 dává 0,250. Hodnoty pod 0,245 značí kalibraci a přesnost zároveň.
Log Loss — logaritmická penalizace overconfidence. Pokud model řekne 95 % a tým prohraje, dostane velkou pokutu (log(0,05) ≈ −3,0). Nižší = lepší. Random model = 0,693 (= ln 2). Model, který je často sebevědomý a často se mýlí, má vysoký log loss i kdyby měl dobrý hit rate.
ROI / Yield — návratnost při flat 1u/zápas. Vsadíš 1 jednotku na každý tip → spočítá se celkový zisk/ztráta a vydělí počtem tipů × 100 %. Příklad: 100 tipů, ztráta −24,1 jednotky → ROI = −24,1 %. To je tvoje skutečná ztráta v jednotkách. Hodnota pod 0 % = prodělek, nad 0 % = zisk.
Bilance 2u stake — peněžní pohled: na každý zápas se sází 2 jednotky. Win = zisk 2 × (kurz − 1), Loss = ztráta −2. Přesně to, co bys vydělal/prodělal v sázkovce při flat sázce 2u.
Avg Edge — průměrný „edge" (kurz sázkovky vs. férový kurz modelu). Kladný edge = sázkovka nabízí vyšší kurz, než je dle modelu spravedlivý. Ale pozor: edge dává smysl jen pokud má model lepší kalibraci než sázkovka — což ukazují Brier a Log Loss výše.
